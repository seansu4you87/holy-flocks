Holy Flocks User Documentation

 

User Interface

 

Overview:

 

Currently, our Holy Flocks UI supports several actions, including adding flock members to the simulation world, starting, pausing, as well as clearing the simulation screen. For debugging purposes, a step button is included to advance the simulation one time step at a time, while a trail range slide connects the previous locations of each flock member in a line.

 

To add a flock member, user can specify the shape of the member and the quantity of members in the flock. Currently the shape only supports circles, but in the future we envision other shapes to be chosen from as well, which the user selects through a dropdown combo box. Likewise, the number of shapes created at one time is currently an integer ranging from 1 to 100. The create button creates the flock into the world. The pause button pauses the simulation, while the start button restarts the simulation if paused. The step button increments the simulation 1 time step at a time; pressing this button while the world is simulating normally will enter the debug incremental simulation mode. Finally, clear button clears the world of all existing flocks.

 

In the future, we want to be able to add multiple flocks within the same simulation panel. We also wish to add particular individuals to a specific flock, or remove members from a flock. We also want to specify which combination of the 3 flocking behaviors are in effect for a certain given flock. In addition, we will add a feature that specifies the role of the window edges, whether they act as walls, wrap around like a torus, or is ignored. Finally, we want to enable the user to load and save specific flocking settings.

 

In terms of code design, the current GUI design is hard-code heavy; refactoring is needed for string constants, but also to redesign interactions with the backend to allow for ease of adding additional features.

Flock Modeling

 

Created new class named Flock

 

Flock contains an ArrayList of Movers.  It has the following methods:

 

public void add(Mover member)

-adds member to the ArrayList of Movers.

 

public void update()

-for each member in the ArrayList of Movers it calls align, cohere(if there are flock members in the annulus), and separate(if there are flock members in the small neighborhood).

 

public void separate(Mover member)

-if there are flock members within a small neighborhood, then the member moves away from those flock members.

 

public void align(Mover member)

-if there are flock members within a large neighborhood, the member moves with the average velocity of these members.

 

public void cohere(Mover member)

-if there are flock members within the annulus (in the large neighborhood but not in the small neighborhood) then the member moves towards the average position of the flock members within the annulus.

 

public ArrayList<Mover> getNeighbors(Mover member, double radius)

-returns an ArrayList of the flock members that lie within the radius of the member's position.

 

public ArrayList<Mover> getAnnulus(Mover member, double minR, double maxR)

-returns an ArrayList of the flock members that lie within the circle of radius maxR but outside the circle of radius minR.
